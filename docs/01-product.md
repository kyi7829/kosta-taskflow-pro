# 01. Product

---

## 목표

> **"지금 누가 뭐 해?"** 라는 질문이 사라지게 만든다.

팀 업무를 시각화하여 누구나 현재 진행 상황을 한눈에 파악할 수 있는 웹앱이다.
Slack 메시지나 구두 보고 없이도 업무 상태를 투명하게 공유한다.

---

## 페르소나

| 항목 | 내용 |
|---|---|
| **대상** | 10명 규모 스타트업의 팀 리더 |
| **연령대** | 30~40대 |
| **상황** | 매일 아침 팀원에게 "어디까지 했어요?" 를 반복적으로 물어야 한다 |
| **불편함** | 업무 현황 파악을 위해 Slack, 회의, 스프레드시트를 번갈아 확인해야 한다 |
| **원하는 것** | 화면 하나로 팀 전체 업무 상태를 즉시 파악하고 싶다 |

---

## MVP 범위

### CRUD 4종 — 모두 화면에서 동작

| 기능 | 설명 |
|---|---|
| **추가(Create)** | 새 업무 생성. 제목, 상태, 마감 시각 입력 |
| **목록(Read)** | 전체 업무 목록 조회. 상태별 분류 표시 |
| **수정(Update)** | 기존 업무 내용 및 상태 편집 |
| **삭제(Delete)** | 업무 삭제. 확인 절차 포함 |

### 상태 분류

업무는 아래 3가지 상태 중 하나를 가진다.

| 상태 | 의미 |
|---|---|
| `todo` | 시작 전 |
| `in_progress` | 진행 중 |
| `done` | 완료 |

### 마감 시각 (due_at)

- 날짜가 아닌 **날짜 + 시간**까지 지정한다.
- 입력 예시: `2026-05-12 18:00`
- 목록에서 마감 시각을 함께 표시한다.
- 마감이 임박하거나 초과한 경우 시각적으로 구분한다.

### 테마 토글

- **라이트 / 다크** 2가지 테마를 지원한다.
- 선택한 테마는 `localStorage`에 저장하여 새로고침 후에도 유지된다.

### 반응형

- 최소 지원 너비: **360px** (모바일 기준)
- 360px에서 레이아웃이 깨지지 않아야 한다.

## UI 톤 — macOS 스타일

| 요소 | 적용 방식 |
|---|---|
| **모서리** | 둥근 모서리 (`border-radius` 적극 활용) |
| **그림자** | 부드럽고 퍼지는 그림자 (`box-shadow: 0 4px 24px rgba(...)`) |
| **카드** | 반투명 배경 (`backdrop-filter: blur`) |
| **폰트** | 시스템 폰트 (`-apple-system, BlinkMacSystemFont, 'Segoe UI', sans-serif`) |
| **전반적 느낌** | 가볍고 정돈된 느낌. 불필요한 장식 없이 콘텐츠 중심 |

## 확장 계획 (MVP 이후)

MVP 완성 후 아래 기능을 순차적으로 추가한다.

| 우선순위 | 기능 | 설명 |
|---|---|---|
| 1 | **JWT 로그인** | 사용자 인증. 팀원별 개인 뷰 제공 |
| 2 | **팀 관리** | 팀 생성, 팀원 초대, 권한 분리 |
| 3 | **Kanban 보드** | 상태 컬럼 기반 드래그앤드롭 뷰 |
| 4 | **팀 채팅** | 업무 단위 댓글 및 멘션 |
| 5 | **CI/CD** | 자동 테스트 및 배포 파이프라인 |

## 범위 외 (MVP에서 구현하지 않는 것)

아래 항목은 MVP 범위에 포함되지 않는다. 요청이 들어와도 즉시 구현하지 않는다.

| 항목 | 제외 이유 |
|---|---|
| 외부 결제 | 수익 모델 미확정 |
| 네이티브 앱 (iOS/Android) | 웹 우선 검증 후 고려 |
| WebRTC | 실시간 영상/음성 통화 — MVP 범위 초과 |
| 외부 캘린더 연동 | Google Calendar 등 — 확장 단계에서 검토 |
| 파일 업로드 | 스토리지 설계 필요 — 확장 단계 보류 |
| WebSocket | MVP는 **폴링** 방식으로 구현. 실시간 기능은 확장 단계에서 도입 |

## 성공 기준

MVP가 완성되었다고 판단하는 조건이다. 아래 기준을 **모두** 충족해야 한다.

| 기준 | 측정 방법 |
|---|---|
| 새로고침 후 데이터 유지 | 페이지 새로고침 시 업무 목록이 그대로 남아 있음 |
| 360px 레이아웃 정상 | 브라우저 너비 360px에서 요소 겹침·잘림 없음 |
| API 응답 200ms 이내 | 로컬 환경 기준 CRUD 각 요청의 응답 시간 |
| CRUD 4종 화면 동작 | 추가·목록·수정·삭제가 모두 UI에서 동작함 |
| 테마 토글 작동 | 라이트↔다크 전환 및 새로고침 후 테마 유지 |
